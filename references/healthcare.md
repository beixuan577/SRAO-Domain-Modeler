# 医疗健康领域模型

## 核心概念词典

| 概念 | 类型 | 定义 | 关键属性 |
|------|------|------|----------|
| 患者 | 实体 | 就诊人员 | 患者ID、姓名、年龄、病史 |
| 生命体征 | 实体 | 实时生理参数 | 心率、血压、血氧、体温 |
| 电子病历(EMR) | 实体 | 患者历史诊疗记录 | 诊断、用药、手术、过敏 |
| 影像(DICOM) | 实体 | CT/MRI/X光影像 | 检查类型、部位、时间、像素数据 |
| 分诊等级 | 实体 | 急诊优先级判定 | 级别(1-5)、依据、判定时间 |
| 辅助诊断 | 实体 | AI诊断建议 | 病灶位置、恶性概率、鉴别诊断 |
| 手术计划 | 实体 | 机器人辅助手术方案 | 路径、配准点、安全边界 |

## 实体关系

```
[患者] --1:N--> [生命体征]
[患者] --1:1--> [电子病历]
[患者] --1:N--> [影像]
[生命体征] --输入--> [分诊等级]
[电子病历] --输入--> [分诊等级]
[影像] --输入--> [辅助诊断]
[辅助诊断] --输入--> [手术计划]
```

## 工作流模板

### WF-MED-001: 急诊分诊

```yaml
trigger: "患者到达急诊"
steps:
  - id: s1
    name: 采集生命体征
    role: 体征设备Agent
    input: 患者接入
    output: 生命体征数值
    next: [s4]
  - id: s2
    name: 语音录入主诉
    role: 语音录入Agent
    input: 患者口述
    output: 结构化主诉
    next: [s4]
  - id: s3
    name: 查询历史病历
    role: EMR查询Agent
    input: 患者ID
    output: 关键病史摘要
    next: [s4]
  - id: s4
    name: 推荐分诊等级
    role: 分诊规则Agent
    input: 体征 + 主诉 + 病史
    output: 分诊等级 + 依据
```

### WF-MED-002: 影像辅助诊断

```yaml
trigger: "DICOM影像接收完成"
steps:
  - id: s1
    name: 接收DICOM
    role: DICOM接收Agent
    input: 影像数据流
    output: 标准化影像
    next: [s2]
  - id: s2
    name: 肺结节检测
    role: 深度学习检测Agent
    input: 影像
    output: 结节位置 + 特征
    next: [s3]
  - id: s3
    name: 恶性概率评估
    role: 风险评估Agent
    input: 结节特征 + 临床信息
    output: 恶性概率 + 建议
    next: [s4]
  - id: s4
    name: 生成报告
    role: 报告Agent
    input: 检测结果 + 风险评估
    output: 结构化诊断报告
```

## 典型KPI

| KPI | 基线 | 目标 |
|-----|------|------|
| 分诊准确率 | 70% | >90% |
| 影像诊断时间 | 30分钟 | <5分钟 |
| 微小病灶检出率 | 60% | >95% |
